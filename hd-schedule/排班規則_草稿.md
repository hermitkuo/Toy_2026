## 0. 名詞與資料來源

| 名詞          | 定義                                  | 來源                            |
| ----------- | ----------------------------------- | ----------------------------- |
| **醫師**      | 簡稱：徐、王、賴、林、郭、超、鄧、翔、祺、禮              | `index.html` PREF_ORDER       |
| **時段**      | 早 / 中 / 晚（每日 3 個）                   | 班表「院內時段」                      |
| **院區 / 班別** | 值班、花蓮HD診、花蓮特約診、玉里、關山、聯安、聯安掛名        | 班表欄位                          |

| **有事（硬）**   | **單一時段不能上** → 該日該時段（早/中/晚）不排，其餘時段照常 | `leave.html` `sessionBusy[]`  |
| **週定義**     | 第 1 週＝含當月 1 號那週，週一為首（週一～週六）         | 現行「找空檔」                       |

> **排班分工（總覽）**：一份完整班表由三個來源合成 ——
> ①**院內HD（早/中/晚）** 由 §0.4 **配額＋優先級** 決定；
> ②**外派（玉里/關山/聯安）與 花蓮OPD/PD** 由**輪值母表**定初步草案（§0.3、S1）；
> ③再疊加 **請假／有事**（H1/H1b）遞補，才是最終班表。

### 0.1 掛名 vs 實際勞務

| 類別             | 欄位                   | 意義                                               |
| -------------- | -------------------- | ------------------------------------------------ |
| **掛名欄（無實際勞務）** | 花蓮HD診、花蓮特約診、聯安掛名     | 僅名義掛名，無特殊勞務。**不計工作量、不影響公平性計算**；仍需填一個名字。但受**掛名檢核**（H8/H9）：掛名者須當天實際在花蓮（非外派/請假），且不與其他掛名重複 |
| **實際勞務欄**      | 值班、院內 早/中/晚、玉里、關山、聯安 | 真正的班，受所有硬限制與公平性規範                                |

> 命名：**花蓮HD診／花蓮特約診** 對應班表資料欄位「花蓮HD」「花蓮特約」（顯示名加「診」；規則文件一律用「診」）。

### 0.2 透析患者分組（院內 HD）

院內 HD 患者依「就診日型 × 時段」固定分組，同一組每週固定同一批患者：

| 日型 | 早 | 中 | 晚 |
|---|---|---|---|
| **W135**（週一/三/五） | W135早 | W135中 | W135晚 |
| **W246**（週二/四/六） | W246早 | W246中 | W246晚 |

共 **6 組**。排某個院內時段＝照顧對應組別患者。→ 供 §3 S4「避免連續安排同一組患者」引用。

### 0.3 花蓮OPD（門診）— 母表週期模板

| 項目       | 內容                                                                                              |
| -------- | ----------------------------------------------------------------------------------------------- |
| **資料來源** | 輪值母表（找空檔 GID 1810798741）中的 **「花蓮OPD」院區列**（另有「花蓮PD」列），非班表欄位                   |
| **粒度**   | 依 **週次（第幾週）× 星期（週一～週六）× 時段（早/中/晚）** 指派；晚時段一般不排（母表為 X）；可多人                 |
| **勞務性質** | **時段勞務**（實際門診）。與「院內 HD 時段」為**兩種不同勞務**，各自獨立計算與顯示                          |
| **週期規則** | 母表所標示月份（目前 **2026/07**）為**規則起始月**；**自起始月起每月沿用**同一「週次×星期×時段」樣式，**除非另有修改**。起始月之前不排 OPD；某月週次超出模板範圍（如第 6 週）時沿用模板最後一週 |

> 用途：`index.html`「我的時段」個人週行事曆據此顯示花蓮OPD 時段。掛名欄（花蓮HD診／花蓮特約診／聯安掛名，§0.1）為全天、無實際勞務，於個人行事曆淡化顯示。

### 0.4 院內HD 月配額（總班數 ＋ 晚班 ＋ 優先級）

院內HD（早/中/晚）**由本節配額＋優先級決定**（不在輪值母表內）。每月席次須**剛好填滿且不外溢**：每個**非週日日 × 3 時段**各 1 人（**週日不排**）。各醫師的月配額設定在設定表「醫師配額」分頁（月份 × 醫師）。

**A. 總班數**

- **當月總格數** ＝（當月天數 − 當月週日數）× 3
- 每人配額方式擇一：
  | 方式 | 參數 | 計算 | 預設 |
  |---|---|---|---|
  | **比率** | 百分比 | 分「餘數」× 該比率 | 徐12／王12／賴12／林20／郭20／鄧24（＝100%，可調） |
  | **每週 N 班** | 每週班數 | N × 當月週數 | 翔 2／超 1／祺 1 |
  | **固定 N 班/月** | 月班數 | 直接採用 N | 禮 9 |
- **餘數** ＝ 當月總格數 − （每週制與固定制者之總和）；再依比率分給比率制醫師
- **取整（保證剛好用完）**：比率制先**無條件捨去**取整；產生的差額（總格數 − 已分配整數和）依 **HD 優先級**由小到大**逐人 +1**，直到加總 ＝ 當月總格數
- **指派順序（院內HD 排到哪天/哪格）**：① 先滿足各人配額席次（硬，剛好填滿）→ ② 依 **S6 偏好**挑日/格 → ③ **S4/S5** 避免連續微調 → ④ 同分時以 **HD 優先級** tie-break
- **缺額處理**：若無法剛好填滿或與硬限制衝突（如當日可用人力不足、掛名無不衝突者），**自動排班頁標示「缺額／警訊」，由人工處理**（不強行亂填）

**B. HD 次數優先級（優先序）**

- **用途**：決定取整餘數（＋1 席）**由誰先吸收**，確保總數剛好用完、每天（非週日）都有人看 HD。優先序數字**越小越先**。
- **預設順序**：`鄧 > 林 > 郭 > 徐 > 王 > 賴 > 翔 > 超 > 祺 > 禮`（比率制、占比大者優先吸收；**可於設定表調整**）
- **定位**：為院內HD 指派的 **tie-breaker（同分時先排誰）**，**非獨立優化目標**；用於「取整補足」與上方「指派順序」第 ④ 關。

**C. 晚班（院內HD 晚，每月固定人頭，可微調）**

- **晚班總數** ＝ 當月非週日天數（每日 1 個晚班）
- **預設人頭**：徐/王/賴/林/郭/鄧/翔 各 **3**；超/祺 各 **2**；禮 **2~3**（由禮與人工微調吸收當月月差）
- 各人**晚班數 ⊆ 總班數**

> 驗算（2026-07）：31 − 4 個週日 ＝ 27 × 3 ＝ **81 格**。每週制＋固定制 ＝ 翔10＋超5＋祺5＋禮9 ＝ 29 → 餘 52 依比率**無條件捨去**（徐6/王6/賴6/林10/郭10/鄧12 ＝ 50），差額 2 依優先級補足（鄧→13、林→11）→ **徐6/王6/賴6/林11/郭10/鄧13 ＝ 52**，全月 81 格填滿。晚班 21＋4＋2 ＝ **27** ✓。
> 用途：`schedule.html`／自動排班頁顯示每人「應排班數（總／晚）」與缺額警訊；本配額為 §3 S3 院內指派與 §2 H3 人力需求的輸入。

## 1. 規則分層（先硬後軟）

排班採「**先滿足所有硬限制，再優化軟限制**」。
硬限制**必須全部成立**；軟限制在硬限制成立的前提下，依 §3 的**權重順位**盡量滿足。

---

## 2. 硬限制（Hard constraints — 一定要成立）

編號即**檢查順位**（上面違反就直接淘汰該排法）。

| # | 規則 | 說明 | 狀態 |
|---|---|---|---|
| H1 | **整天請假不排** | 醫師 `fullDayLeave[]` 標記的**整天**，該日所有院區/時段一律不得被排 | ✅ 明確 |
| H1b | **有事時段不排** | 醫師 `sessionBusy[]` 標記的**該日該時段（早/中/晚）**不得被排；其餘時段照常可排 | ✅ 明確 |
| H2 | **不分身（含例外）** | 同一醫師、同一日、同一時段只能在一個院區。**例外**：被排 玉里/關山/聯安（日間外派）者，**可同時再排「院內晚」**（外派回來上晚班）。掛名欄不受此限（§0.1） | ✅ 明確 |
| H3 | **人力需求** | 每個院區 × 時段所需人數必須被填滿（見 §4：院內/外派各營業時段**各一人**；花蓮OPD 依母表） | ✅ 明確 |
| H4 | **資格＝母表／配額＋遞補** | 外派（玉里/關山/聯安）與 花蓮OPD/PD 預設依**輪值母表**；**院內HD 依 §0.4 配額**。**預設者當日請假或有事，改由其他有空的人遞補**（有空＝該時段未被排且非請假/有事） | ✅ 明確 |
| H5 | **值班輪值** | 值班由 **徐、王、賴、林、郭、鄧、翔** 7 人輪流（**不含 超、祺、禮**），**一人一天**；**遇週六、週日由同一人連值兩天**。輪值順序**跨月延續、不重置**；遇特殊情境可調動（該人請假不在、或**內科總值班同步值班**）。**值班的人當天仍可被排外派班**（值班不獨占當日）。無「值班隔日不排」等額外連動限制 | ✅ 明確 |
| H6 | **外派連續性** | **玉里、關山：整天同一人**（不可拆早/中給不同人）；**聯安：情況允許就同一人**（軟性偏好，見 S4） | ✅ 明確 |
| H7 | **聯安掛名連動** | **聯安掛名 ＝ 聯安 同一人**；**除非** 聯安 為 翔/祺/超/禮 之一，此時聯安掛名為另一人 | ✅ 明確 |
| H8 | **花蓮HD診 掛名檢核** | 花蓮HD診 掛名者**不可**與當日 **聯安掛名／玉里／關山／請假** 為同一人（該人已外派或請假、或已占用另一掛名）。**若找不到不衝突者 → 將花蓮HD診 拆成早/中/晚**，逐時段各找不衝突的人 | ✅ 明確 |
| H9 | **花蓮特約診 掛名檢核** | 花蓮特約診 掛名者**不可**與當日 **聯安掛名／玉里／關山／請假** 為同一人。**若衝突 → 將花蓮特約診 拆成早/中/晚**，逐時段各找不衝突的人 | ✅ 明確 |

> ✅ **無單月上限**：不設每人外派上限、連續上班天數上限。
> 註：H8/H9 的「請假」指整天請假；拆成早/中/晚後，各時段亦應避開該時段「有事」者。

## 3. 軟限制（Soft constraints — 依順位優化）

演算法對每個候選排法計分，**順位越前、權重越高**。

> 註：「請假」「有事」皆為硬限制（H1/H1b），不在此層。
> **重要（分工）**：外派/OPD 由**母表**結構性決定、值班由 **H5** 輪值；**院內HD 由 §0.4 配額＋優先級**決定。整體流程＝母表定外派/OPD 草案 → 院內HD 依配額填席（配額 → S6 → S4/S5 → 優先級 tie-break）→ 請假/有事遞補。

| 順位 | 規則 | 內容 | 狀態 |
|---|---|---|---|
| S1 | **外派安排** | **依「找空檔」輪值母表規則**安排（母表已內含外派公平/輪替） | ✅ 依母表 |
| S2 | **值班安排** | **徐、王、賴、林、郭、鄧、翔 輪流，一人一天，遇週六日同一人連兩天**（＝ H5，母表外的固定輪值） | ✅ 明確 |
| S3 | **院內HD 席次** | **依 §0.4 配額（總班數＋優先級）填滿**；母表決定當日可用人力（被外派/OPD 占用同時段者不可再上院內同時段，H2） | ✅ 依 §0.4 |
| S4 | **避免連續（院內＋同組患者）** | 院內時段**避免同一醫師連續**；且**避免連續安排同一組患者**（§0.2 六組之一）。**外派：依母表規則**，不受此限 | ✅ 明確 |
| S5 | **連續規則的範圍** | S4 的院內「避免連續」**僅適用週一～週五**；**週六例外**（週六可連續安排） | ✅ 明確 |
| S6 | **醫師偏好** | 收集**「星期幾 × 院內時段（早/中/晚）」的優先偏好**（例：優先 週一早、週三中）。以表格輸入，排院內HD 時（§0.4 指派順序第②關）在不破壞硬限制下盡量滿足 | ✅ 結構已定（待建表收集） |
| S7 | **花蓮HD診／花蓮特約診 分人** | 兩個掛名**盡量不要是同一人**（分給不同醫師）；此為軟性，硬性檢核見 H8/H9 | ✅ 明確 |
| S8 | **掛名診 指派池與平均** | **花蓮HD診**掛名由 **徐、王、賴、林、郭、鄧** 擔任、以**平均**為原則；**花蓮特約診**掛名由 **賴、林、郭、鄧** 擔任、以平均為原則。兩者皆**優先選當日不與其他欄位衝突者**（呼應 H8/H9：不與 聯安掛名／玉里／關山／請假 同人）。**平均為軟性，無法平均可接受**；衝突檢核（H8/H9）為硬性優先 | ✅ 明確 |

## 5. 演算法路線（Task 3 預覽，先不實作）

公平性大多已由**母表 + 值班輪值 + §0.4 配額**決定，故流程偏「套來源 → 補洞 → 微調」：

1. **讀入**：輪值母表（外派 + 花蓮OPD/PD，GID 1810798741）＋ 值班輪值（H5）＋ **醫師配額（§0.4）** ＋ **S6 偏好** ＋ 各醫師 `fullDayLeave[]`/`sessionBusy[]`（`leave.html`）＋ §4 需求。
2. **母表草案**：依 S1 產生外派（玉里/關山/聯安）與 花蓮OPD/PD 初步指派；依 S2/H5 產生值班輪值。
3. **院內HD 填席**：依 **§0.4 配額**（總班/晚班）決定各人席次，指派順序＝**配額(硬) → S6 偏好 → S4/S5 避免連續 → HD 優先級 tie-break**。母表占用者（外派/OPD 同時段）不可上院內同時段（H2；外派者可上院內晚）。
4. **遞補（H1/H1b/H4）**：請假/有事者 → 換其他有空的人；套用 H2 晚班例外、H6/H7 連動。
5. **掛名檢核（H8/H9/S7/S8）**：花蓮HD診／花蓮特約診 掛名依 S8 池與平均、不衝突優先；衝突則拆早/中/晚逐時段補；並依 S7 讓兩者不同人。
6. **輸出**：對齊現行 `SHIFT_COLS` 的班表 JSON（掛名診若拆分則帶早/中/晚）→ 回填設定表「自動排班草稿」分頁或前端顯示。
7. **缺額 / 人工覆核**：無法填滿或衝突 → **自動排班頁標示「缺額／警訊」** → 人工微調定案；並列「未滿足的軟規則清單」。

## 6. 決策紀錄與現況

規則面主體已定，隨需求增修（見文末 changelog）。摘要：

- **硬限制 H1–H9**：請假(整天)、有事(單時段)、不分身+晚班例外、人力需求(各一人)、資格(母表/配額+遞補)、
