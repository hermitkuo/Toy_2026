排班決策不僅是填補空缺，更涉及人力安全、服務量、公平性、個人限制及突發彈性等多方面權衡。針對血透室或多院區支援情境，排班決策可分為以下八類：

### 1. 人力安全決策：確保關鍵時段人力充足
這是最核心的 P0 決策，旨在識別並避免人力不足的風險。

#### 決策考量
*   **請假限制**：特定日期（如週三全天）人力偏低時，是否應限制請假？
*   **安全人力門檻**：當外派、門診、請假同時發生時，院內血液透析（HD）人力是否低於最低安全標準？
*   **時段缺口**：早、中、晚班哪個時段最容易出現人力缺口？
*   **連假前後規劃**：連假前後是否需要預先增補人力或禁止換班？

#### 對應圖表
週間人力熱力圖（heatmap）、日曆熱力圖（calendar heatmap）、風險關鍵績效指標（KPI）卡。

#### 決策輸出範例
*   「週三全天可用人力 < 3 人時，不允許再核准請假。」
*   「若同日外派 ≥ 3 人且門診 ≥ 2 時段，需指定備援人力。」

### 2. 公平性決策：平衡工作量分配
此類決策旨在客觀化工作量分配，避免主觀爭議。

#### 決策考量
*   **年度總工作量**：誰的年度總工作量遠高於平均？
*   **外派負擔**：誰的外派負擔過重？
*   **特殊班次**：誰的假日、大抽、週六班次偏多？
*   **班次組合負荷**：誰的門診 + HD + 外派組合，實際負荷比表面班數更重？
*   **特殊醫師計算**：新進或部分時段醫師的工作量是否應分開計算？

#### 對應圖表
發散式條形圖（diverging bar）、堆疊條形圖（stacked bar）、棒棒糖圖（lollipop plot）、個人負荷卡。

#### 關鍵考量
不應只看「班數」，而應考量「加權工作量」。例如，外派一天、院內早班、門診一節，其負荷應有不同權重。

### 3. 請假核准決策：建立請假審核標準
排班系統最實用的功能之一，用於輔助請假審核。

#### 決策考量
*   **人力充足性**：醫師請假後，該日最低人力是否仍足夠？
*   **空窗風險**：是否會造成同院區、同時段、同技能組的人力空窗？
*   **請假集中度**：同一週是否已有過多請假？
*   **高風險時段**：請假日期是否落在春節、清明、端午、中秋、暑假等高風險區？
*   **代理人指定**：是否需指定代理人後才能准假？

#### 對應圖表
日曆熱力圖（calendar heatmap）、每日風險徽章（badge）、請假熱門週條形圖。

#### 建議準假燈號
*   **綠燈**：可准。
*   **黃燈**：可准，但需指定代理人。
*   **紅燈**：不建議准，除非同時完成換班。
*   **灰燈**：資料不足，例如未來月份院內 HD 尚未排完。

### 4. 換班決策：評估換班對整體排班的影響
此為從儀表板進階到決策支援的關鍵功能。

#### 系統可回答問題
*   **公平性改善**：A 與 B 換班後，公平性變異係數（CV）是否下降？
*   **人力風險變化**：該日人力風險是否改善或惡化？
*   **連續工作天數**：是否造成某人連續工作天數過長？
*   **週末/假日班次**：是否讓某人週末／假日班次偏高？
*   **院區熟悉度/技能需求**：是否影響院區熟悉度或指定技能需求？

#### 最佳介面呈現
不只顯示「可否交換」，更應顯示「交換前後差異卡」，例如：
*   交換前：A +12、B -8。
*   交換後：A +8、B -4。
*   整體公平性改善 18%。
*   但週三早班人力從 4 降到 3，需確認備援。

### 5. 院區支援決策：優化多院區人力分配
多院區支援涉及地點、移動成本、熟悉度、時段等特殊考量。

#### 決策考量
*   **依賴度**：某院區是否過度依賴少數醫師？
*   **外派集中度**：某醫師是否外派過度集中？
*   **固定/輪替池**：玉里、關山、聯安等院區是否需要固定或輪替支援池？
*   **連續外派**：是否應避免同一人連續外派？
*   **資深度/熟悉度**：某些院區是否需要特定資深度或熟悉度的醫師？

#### 對應圖表
堆疊條形圖（stacked bar）、桑基圖（Sankey / alluvial）、院區 × 醫師熱力圖（heatmap）。

#### 建議指標拆分
*   **班數**：支援次數。
*   **負荷**：依距離、時段數、交通與責任加權後的工作量。
    *   此可避免遠距外派負荷被低估。

### 6. 班種組合決策：實現透明且合理的班種分配
不同醫師的班種組合應有所差異，但需保持透明。

#### 決策考量
*   **主要負荷來源**：誰的工作負荷主要來自門診？誰主要被外派佔用？誰院內 HD 過重？
*   **高摩擦班種**：是否有醫師被安排過多高摩擦班種？
*   **上限/權重設定**：是否需要設定每人外派上限或門診補償權重？

#### 對應圖表
堆疊條形圖（stacked bar）、個人組成卡、月份組成小倍圖（small multiples）。

#### 盲點提醒
僅以總工作量排名可能忽略「工作性質」差異。例如，院內 HD、外派、門診、週六大抽等，即使分數相同，體感負擔也不同。

### 7. 未來排班補洞決策：預先識別並填補人力缺口
此為最接近排班工作流程的決策。

#### 系統可回答問題
*   **未排班日期**：下個月哪些日期尚未排完整？
*   **未注意缺口**：哪些時段缺人但尚未被注意？
*   **院區覆蓋**：哪些院區尚未覆蓋？
*   **優先補班人選**：哪些醫師本月負荷偏低，可以優先補班？
*   **不應補班人選**：哪些醫師已超載，不應再補？

#### 對應圖表
缺口清單、日曆熱力圖（calendar heatmap）、可用人力矩陣、推薦候選人排序。

#### 建議補洞推薦功能
*   **缺口**：7/15 週三早班，院內 HD。
*   **候選 1**：王醫師，可用，月負荷 -5，無連班。
*   **候選 2**：祺醫師，可用，月負荷 -3，但前一天外派。
*   **不建議**：鄧醫師，本月已 +9 且週六班偏多。

### 8. 規則設計決策：平衡固定規則與人工彈性
排班系統最終會面臨治理問題，需平衡規則的嚴謹性與現場的彈性。

#### 建議固定的規則
最低安全人力、請假上限、高風險日准假條件、外派加權、週末／假日班統計、連續工作天數上限、未來月份資料不完整標示。

#### 建議保留人工判斷的規則
特殊家庭因素、臨時醫療任務、教學、會議、研究責任、特定院區熟悉度、資深醫師臨床備援、新進醫師逐步承擔負荷。

#### 務實的排班系統發展路徑
不應追求「全自動最佳化排班」，因醫療現場限制與例外過多，且公平性的社會接受度比數學最佳解更重要。建議路徑為：
1.  **風險提示**
2.  **交換前後模擬**
3.  **候選人排序**
4.  **最後才考慮最佳化演算法**

### 優先排班決策模組
若要將現有統計頁升級為排班決策頁，建議優先開發以下五個模組：

| 優先級 | 決策模組       | 直接回答的問題             |
| :---- | :------------- | :------------------------- |
| P0    | 低人力風險日偵測 | 哪些天不能再請假／換班？   |
| P0    | 請假核准燈號   | 這個假能不能准？           |
| P0    | 公平性偏差排行 | 誰已經太重，誰可以補？     |
| P1    | 換班前後模擬   | A/B 交換後變好還是變壞？   |
| P1    | 補洞候選人推薦 | 這個空班最適合找誰？       |

核心判斷是：統計圖表應服務決策，而非僅展示統計。對血透室排班而言，最有價值的不是多一張漂亮圖，而是讓主管在 30 秒內知道：「哪一天危險、誰太累、這個假能不能准、這個洞該找誰補。」

### 營運儀表板升級建議
現有班表統計頁已具備統計分頁、年度切換、月份熱區圖、院內 HD、院區支援、週間人力吃緊、大抽週五／週六、請假熱門區間、總工作量等模組，且資料來自 Google 試算表同步。這表示基礎架構已適合做成「營運儀表板」，而非單純增加統計圖。

#### 一、建議保留但升級：熱力圖（Heatmap）應具備更多語義
目前大量使用熱力圖是正確的，因班表資料本質是「人 × 月」、「時段 × 星期」、「人 × 工作量」矩陣。建議升級為三種不同語義的熱力圖，而非僅使用同一套色階：

1.  **負荷熱力圖**：顯示實際班數、總工作量，回答「誰比較累」。
2.  **公平性熱力圖**：每格顯示「與同儕平均差距」（例如 +2.5、-1.0），而非原始班數。這比單純總計更適合討論排班公平性。
3.  **風險熱力圖**：例如週三全天平均可用人力 2.2，不僅數字標紅，更標示「低於安全門檻」。這將圖表從描述統計升級為管理工具。

不建議增加過多不同顏色的熱力圖，以免使用者誤以為每張圖語義不同，而實質僅是裝飾差異。

#### 二、加入「公平性指標」：提升管理價值
班表統計最常引發爭議的是「是否吃虧」，而非總量。建議新增一個「公平性面板」，置於統計分頁最上方。

##### 建議指標與圖表
| 指標             | 圖表建議             | 管理意義             |
| :--------------- | :------------------- | :------------------- |
| 年度總工作量     | 排名橫條圖           | 誰整體負荷最高       |
| 與平均差距       | 發散式條形圖         | 一眼看出超載／低載   |
| 週末與假日負荷   | 棒棒糖圖             | 比一般條形圖更適合少量類別 |
| 夜間／大抽次數   | 點圖（Dot plot）     | 避免柱狀圖過度厚重   |
| 吉尼係數（Gini coefficient）/ 變異係數（CV） | 單一 KPI 卡          | 用一個數字表示公平程度 |

其中最推薦發散式條形圖（diverging bar chart）：以平均值為 0，中線右邊表示高於平均，左邊表示低於平均。這比「總計 289 vs 276」更能讓主管快速判斷誰偏重。

#### 三、加入「小倍圖（Small Multiples）」：同一邏輯看不同醫師
若想讓統計圖更多元但不混亂，小倍圖是最佳選擇。
例如，每位醫師一張小折線圖，顯示 1–12 月總工作量，所有醫師共用同一個 Y 軸範圍。這樣可以直接看出：
*   誰是穩定高負荷。
*   誰是某幾個月工作量爆增。
*   誰是新加入或後段才開始排班。
*   哪幾個月整體人力明顯吃緊。

這比將 10 位醫師全部塞進同一張多線圖更佳。多線圖在班表場景通常會變成「義大利麵圖」，不值得投入。

#### 四、加入「日曆式熱力圖（Calendar Heatmap）」：呈現請假與人力風險
目前已有年度請假熱門區間的週柱狀圖，但若想進一步直覺化，建議新增日曆式熱力圖。

##### 用途
1.  **請假集中度**：查看全年哪些日期請假集中。
2.  **節假日疊加**：疊加國定假日、連假、週末資訊。
3.  **高風險日標示**：標示「同日請假人數 ≥ N」或「可用人力低於門檻」的高風險日。

這種圖對排班最有價值，因為班表決策不僅看月份，更看「哪一天會爆」。現有資料已包含非週末國定假日 HD 值班與請假週分佈，日曆熱力圖可整合這些資訊。

#### 五、加入「桑基圖（Sankey / Alluvial）」：僅限於院區流向，避免濫用
若想使用較新的視覺化元素，可考慮桑基圖或沖積圖（Alluvial plot），但僅建議用於一個場景：醫師人力從「醫師 → 院區 → 時段」的流動。
例如：
*   醫師 → 玉里／關山／聯安 → 早／中／晚
*   醫師 → 院內 HD／外派／門診 → 加權工作量
*   月份 → 院區 → 醫師

這非常適合回答：「人力到底被哪些院區消耗？」但不建議將桑基圖用於一般班數統計。桑基圖雖然美觀，但若無「流量」或「來源到去向」的語義，將成為視覺噪音。

#### 六、不建議加入「風險雷達圖」
雷達圖常被用於多維比較，但對班表公平性而言效果不佳，原因如下：
*   **尺度不一致**：值班次數、門診、外派、假日班、夜間班等不同維度，難以在同一個圓形面積中比較。
*   **精準度不足**：雷達圖雖具設計感，但難以精準比較。
*   **圖表混亂**：一旦醫師超過 4–5 人，圖表將嚴重混亂。

若確實需要多維個人檔案，建議改用「個人負荷卡片」：
*   值班：高 / 中 / 低
*   院內 HD：高 / 中 / 低
*   外派：高 / 中 / 低
*   假日：高 / 中 / 低
*   請假衝突風險：高 / 中 / 低

這比雷達圖更適合臨床排班討論。

#### 七、加入「預警型圖表」：從統計頁轉變為決策頁
最值得升級的是將描述統計轉變為「排班預警」。建議新增一個「班表風險摘要」區塊，置於統計頁最上方。

##### 預警內容與視覺化方式
| 預警                 | 視覺化方式           | 例子                   |
| :------------------- | :------------------- | :--------------------- |
| 可用人力低於門檻     | 紅色徽章 + 迷你熱力圖 | 週三全天 < 3 人        |
| 某醫師月負荷超過平均 + 2 個標準差 | 異常點標記           | 鄧醫師 7 月工作量 40   |
| 連假前後請假集中     | 日曆熱力圖           | 6/1–6/7 請假 27 人次   |
| 院區支援過度集中     | 帕累托圖（Pareto chart） | 前 3 人承擔 50% 外派   |
| 未來月份資料不完整   | 灰階或虛線           | 8–12 月院內 HD 尚未完整 |

這是最實用的方向。使用者不僅想看圖，更想知道「哪裡需要處理」。

#### 八、建議圖表組合順序
1.  **第一層（主管 10 秒決策）**：KPI 卡片，顯示總工作量最高者、最低者、公平性 CV、最吃緊時段、最高請假週。
2.  **第二層（營運診斷）**：發散式條形圖、日曆熱力圖、週間時段熱力圖、院區支援堆疊條形圖。
3.  **第三層（個人追蹤）**：點選醫師後顯示個人迷你折線圖（sparkline）、小月曆、班種組成。
4.  **第四層（排班修正）**：顯示「若把某班從 A 換到 B，公平性指標改善多少」。這將從儀表板進化為決策支援。

#### 九、具體建議新增的圖表清單

| 優先級 | 圖表                 | 用途                 | 是否值得做 |
| :---- | :------------------- | :------------------- | :--------- |
| P0    | 發散式條形圖         | 顯示每人與平均工作量差距 | 很值得     |
| P0    | 日曆熱力圖           | 查看每日請假／人力風險 | 很值得     |
| P0    | KPI 風險卡片         | 直接指出異常與風險   | 很值得     |
| P1    | 小倍圖折線圖         | 每位醫師月趨勢       | 值得       |
| P1    | 堆疊條形圖           | 工作量組成：院內／外派／門診 | 值得       |
| P1    | 棒棒糖圖             | 假日、大抽、週末班比較 | 值得       |
| P2    | 桑基圖／沖積圖       | 醫師→院區→時段流向   | 可做，但要克制 |
| P2    | 盒形圖（Boxplot）／小提琴圖（Violin plot） | 比較各醫師月負荷分佈 | 進階使用者才需要 |
| 不建議 | 圓餅圖               | 班表組成比例         | 多數情境不如條形圖 |
| 不建議 | 雷達圖               | 多維公平性           | 容易誤導   |
| 不建議 | 3D 圖表              | 裝飾性高、判讀差     | 不值得     |

#### 十、建議採用的設計原則
現有頁面已具備良好的卡片式結構與熱力圖基礎。下一步不應追求「炫」，而是專注於三件事：
1.  **明確圖表問題**：讓每張圖回答明確問題，例如：誰累、哪天危險、哪個院區消耗人力、哪個人被不公平分配。
