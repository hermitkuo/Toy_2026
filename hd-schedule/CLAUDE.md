# hd-schedule — 專案規則與設計取向

本檔記錄「從程式碼本身推不出來」的設計決策，供日後 Claude Code 續作時遵循。
排班的臨床規則（硬限制 H1–H9／軟限制 S1–S7）另見 `排班規則_草稿.md`；本檔只放產品/UI 取向。

## 列印（index.html「找班表」頁 `@media print`）

- **醫師縮寫的可讀性 ＞ 整月硬塞一頁。** 行政人員實際拿列印/PDF 的班表在用，字太小就沒法用。
  - 醫師縮寫維持約 **14px 加粗**；正常月份 1 頁、排班滿版時**自動延伸到第 2 頁**。
    不可為了「單頁」把字縮到 ~10px（此路已試過並否決）。
  - 直式 A4；**彩色輸出但保持黑白友善**（常被列印成 PDF，也常黑白影印）。
  - 列印時 `beforeprint` 暫切亮色主題輸出、`afterprint` 還原使用者主題；`table-layout:fixed` 確保欄寬不超頁、名字自動換行。
