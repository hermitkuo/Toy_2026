總結判斷

AI agent orchestration 的成熟方向，不是讓多個 agent 自由群聊，而是把 LLM 放進可控的 workflow runtime。Harness Engineer 與 Loop Engineer 是把這些 orchestration pattern 產品化的兩個職能切面。

Orchestration = 流程拓撲；Harness = 安全執行外殼；Loop = 閉環推進機制；Evaluation = 品質與風險量測；Platform = 規模化營運。

最穩基礎

Deterministic workflow / DAG。能用 rule、SQL、function 解決的，不要交給 LLM 自由推理。

最有生產價值

Graph-based orchestration + durable execution + validator。讓 agent 有狀態、中斷點、回放、審計軌跡。

最適合醫療

Agentic subroutine 放在受控節點內；臨床決策與醫囑寫回必須 human-gated。

Harness Engineer 與 Loop Engineer

這兩個詞不應被理解成固定職稱，而是 agent 系統產品化時的兩組能力。小團隊可由同一人負責；醫療級 production 最好拆分治理責任。

Harness Engineer

負責 agent 活在什麼環境裡。

  • Tool registry / MCP integration
  • Context pipeline / RAG grounding
  • Guardrails / policy enforcement
  • Schema validation / PHI boundary
  • Trace / audit / replay
  • Eval harness / human approval workflow

本質是把模型能力變成可觀察、可權限控管、可驗證、可追責的系統行為。

Loop Engineer

負責 agent 如何反覆推進、修正與停止。

  • Trigger / event design
  • Task decomposition
  • Success criteria
  • Validator / verifier
  • Retry / repair / fallback strategy
  • Stop condition / escalation policy

本質是設計閉環控制系統：不是讓 agent 一直跑，而是知道何時繼續、換策略、升級或停止。

Prompt Engineer → Context Engineer → Harness Engineer → Loop Engineer → Agent Platform / Reliability Engineer

醫療 AI 範例：透析 Anemia Management Agent

這個例子說明為什麼醫療場景不能只做「聰明 agent」，而要做 harness + loop + human gate。

Harness 設計

  • 資料來源：Hb trend、Ferritin、TSAT、ESA dose、IV iron、住院/感染/出血 flag、local protocol。
  • 工具：read-only EHR query、lab trend summarizer、protocol retriever、dose simulator、note draft generator。
  • 安全邊界：不可直接改醫囑；Hb < 7、Hb > 12、active bleeding、recent admission 強制醫師審核。
  • 所有建議必須附資料 timestamp、protocol source、可追蹤理由。

Loop 設計

  • Trigger：每週一早上，或新 Hb result 回來後觸發。
  • Action：抓資料 → eligibility → 產生建議 → 驗證 → 送 review queue。
  • Verification：資料完整性、lab 是否過期、protocol 是否違反、contraindication。
  • Stop：建議完成、資料不足、高風險人工判斷、retry 超過上限。

建議團隊 Roadmap

Layer 1 — Harness

建立 agent 的安全執行外殼：tools、permissions、RAG、memory、guardrails、trace、eval、audit。

Layer 2 — Loop

建立可控閉環：trigger、goal、validator、repair、retry、stop、human escalation。

Layer 3 — Clinical Workflow

接到真實醫療流程：review queue、採納/拒絕原因、protocol update、quality dashboard。

最小可行版本

每日或每週觸發 → 抓取病人清單 → 找出 protocol deviation / high-risk cases → 產生可解釋摘要 → validator 檢查資料完整性與規則 → 放入醫師 review queue → 記錄採納、修改、拒絕原因 → 用結果更新 eval dataset
職稱建議： 不必急著對外招聘「Harness Engineer」或「Loop Engineer」。比較容易被醫療主管、資安與法遵理解的名稱是 Agent Orchestration / Reliability Engineer 或 Clinical Agent Platform Engineer。

Not-to-do：不值得投入的方向

不要做純 swarm / 群聊式 multi-agent

看起來有智慧，實際上難除錯、難審計、成本高，臨床環境尤其不適合。

不要把所有流程 agent 化

能用 SQL、rule engine、function、workflow 解決的，就不要交給 LLM。這是工程紀律，不是保守。

不要無治理接一堆 MCP server

工具越多，攻擊面越大。醫療場景應採白名單、最小權限、dry-run、審計與輸出驗證。

不要追求 fully autonomous clinical loop

醫療產品競爭力不是不需要醫師，而是讓醫師在正確時間看到可驗證建議，並留下審計軌跡。

成熟的 agent 系統，不是讓 AI 擁有更多自由，而是讓 AI 在更清楚的邊界內產生更高密度、可追責的工作成果。
